CSS Color Reference

The benefits and disadvantages between Color Name, Hex, RGBA, HSLA and HWB

Color Name

The first way to define a color for your website is through the use of Color Names. These are simple words that will define the color used on the site.
The disadvantage to this method is the fact you are stuck with a predefined list of color names that you cannot add to.

It's imporant to remember, all these colors can be accessed through the use of CSS keyword color: or background-color:

Examples below...

Green Red Teal
green; red; teal;

HEX

The second way to define a color for your website is through the use of Hexadecimal Colors. These are parameters that will define the color used on the site as specified using #RRGGBB. The integers range between 00 through to FF being the highest intensity. Another thing to remember is that if the color uses the same value for Red, Green or Blue, eg. #ff0000 which gives us the color Red, we can shorten this to #f00.

It's imporant to remember, all these colors can be accessed through the use of CSS keyword color: or background-color:

Examples below...

Green Red Teal
#008000 #ff0000 or #f00 #008080

RGBA

The third way to define a color for your website is through the use of RGBA. These are parameters that will define the color used on the site based on the amount of Red, Green and Blue light sources, with the intensity ranging from 0 (none) all the way to 255 (full). The Alpha , which ranges from 0.0-1.0 will specify the opacity for the chosen color, 0.0 being transparant and 1.0 being opaque. If not being used, we can omit the Alpha as in the examples below. This would represent a RGB color code.

It's imporant to remember, all these colors can be accessed through the use of CSS keyword color: or background-color:

Examples below...

Green Red Teal
rgb(0,128,0) rgb(255,0,0) rgb(0,128,128)

HSLA

The fourth way to define a color for your website is through the use of HSLA. These are parameters that will define the color used on the site based on the amount of Hue, Saturation and Lightness, with the Hue being an angle of the color circle between 0 and 360 degrees. The Saturation and Lightness are percentages between saturated to grey and white to black. The Alpha , which ranges from 0.0-1.0 will specify the opacity for the chosen color, 0.0 being transparant and 1.0 being opaque. If not being used, we can omit the Alpha as in the examples below. This would represent a HSL color code.

It's imporant to remember, all these colors can be accessed through the use of CSS keyword color: or background-color:

Examples below...

Green Red Teal
hsl(120, 100%, 25%) hsl(0, 100%, 50%) hsl(180, 100%, 25%)

HWB

The fifth way to define a color, which is currently not used, but suggested as the standard for CSS4 is HWB. These are parameters that will define the color used on the site based on the amount of Hue, Whiteness and Blackness, with the Hue being an angle of the color circle between 0 and 360 degrees. The Whiteness and Blackness are percentages.

It's imporant to remember, all these colors can be accessed through the use of CSS keyword color: or background-color:

The examples below will not work yet, but are shown purely to show the syntax needed to use when HWB becomes available...

Green Red Teal
hwb(120, 0%, 50%) hwb(0, 0%, 0%) hwb(180, 0%, 50%)
